## What is Hugo
Hugo was created using the Go programming language by Steve Francia in 2015 and is now being developed together with Bjørn Erik Pedersen and other contributors. Hugo is similar to [jekyll](https://jekyllrb.com/) and [11ty](https://www.11ty.dev/). Due to its use of Go, Hugo's performance is very high; it can generate a page in just milliseconds. Imagine having a blog with over 10,000 posts, and it only takes about 2 seconds to build! Moreover, the output is HTML files, so there's no need to pay for hosting space because GitHub Pages is sufficient.

However, not only because of its performance, Hugo has a powerful Content Management System (CMS) feature. You only need to write in markdown format, and Hugo will automatically convert it to HTML files with dynamic data inside.

# Why I Choose Hugo?
1. Lightweight and fast
   Nowadays, web and application development is not just about making it work, but now we have to pay attention to the quality and performance of the web/application we create, this is due to the development of technology.
2. Outputs static files
   With static file output, it makes it easy for developers to host the website, plus no complicated server-side configurations are needed.
3. Good and growing community
   Currently, Hugo has been used by at least 1.3k developers, with 71.2k stars on its [GitHub repository](https://github.com/gohugoio/hugo).
4. Many free hosting options
   Yes, because it's static, you can host it anywhere, even GitHub Pages alone is enough, but if you're not familiar with [github actions](https://github.com/features/actions), you can use alternatives like [netlify](https://www.netlify.com/), [vercel](https://vercel.com/), and [cloudflare pages](https://pages.cloudflare.com/).
